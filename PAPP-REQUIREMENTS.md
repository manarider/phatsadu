# PAPP Requirements

## 1. ชื่อระบบ
**ระบบบริหารจัดการพัสดุ เทศบาลนครนครสวรรค์ (PAPP)**

## 2. วัตถุประสงค์
พัฒนา Web Application สำหรับบริหารจัดการ:
- ครุภัณฑ์
- วัสดุ
- งานแจ้งซ่อมครุภัณฑ์
- งานบันทึกรายการซ่อม
- งานสต๊อกและเบิกจ่ายวัสดุ

## 3. เทคโนโลยีหลัก
- Nginx
- NodeJS
- React
- PostgreSQL

## 4. URL การใช้งาน
ระบบต้องเปิดใช้งานผ่าน:
- `http://app.nsm.go.th/phatsadu/`
- `http://192.168.100.152/phatsadu/`

Nginx ทำหน้าที่รับ request ที่ port 80 และส่งต่อไปยัง NodeJS backend ที่ port 3033

## 5. Authentication
- ใช้ระบบ UMS กลางที่ `http://192.168.100.151/ums/`
- ระบบนี้ไม่เก็บรหัสผ่านผู้ใช้
- ใช้ข้อมูลสิทธิ์จาก UMS และ map role เข้ากับระบบ PAPP

## 6. โมดูลหลัก

### 6.1 ระบบบริหารจัดการครุภัณฑ์
รองรับ:
- ค้นหา
- เพิ่ม
- แก้ไข
- ดูรายละเอียด
- แจ้งซ่อม
- บันทึกประวัติซ่อม

#### ข้อมูลสำคัญที่ต้องจัดเก็บ
- รหัสครุภัณฑ์ `XXX-XX-XXXX`
- Serial Number
- ประเภทครุภัณฑ์
- รายการครุภัณฑ์
- หน่วยงานเจ้าของครุภัณฑ์
- วันที่ได้มา
- ราคา
- ตำแหน่งที่ตั้ง
- ผู้ดูแล
- สถานะ
- ภาพถ่ายครุภัณฑ์
- รายละเอียดครุภัณฑ์
- โครงการ
- หมายเหตุ

#### สถานะครุภัณฑ์
- ใช้งานได้
- ใช้งานไม่ได้
- จำหน่ายแล้ว

#### ค่ามาตรฐานที่ใช้ในระบบ (Canonical Enum)
เพื่อให้เอกสารตรงกัน ให้ใช้ค่าใน API/Database ดังนี้:
- `ACTIVE` = ใช้งานได้
- `INACTIVE` = ใช้งานไม่ได้
- `DISPOSED` = จำหน่ายแล้ว

#### เงื่อนไขรูปภาพ
- ขนาดไม่เกิน 2MB ต่อภาพ
- ไม่เกิน 4 ภาพต่อครุภัณฑ์ 1 รายการ

### 6.2 ระบบบริหารจัดการวัสดุ
รองรับ:
- ค้นหา
- เพิ่ม
- แก้ไข
- รับเข้า
- เบิกจ่าย
- ตรวจสอบสต๊อก
- ปรับปรุงยอด

#### ข้อมูลสำคัญที่ต้องจัดเก็บ
- ประเภทวัสดุ
- รายการวัสดุ
- หน่วยงานเจ้าของวัสดุ
- วันที่ได้มา
- ราคา
- จำนวน
- หน่วยนับ
- จุดต่ำสุดของสต๊อก
- หมายเหตุ

## 7. Master Data

### 7.1 Departments (หน่วยงาน)
1. สำนักปลัดเทศบาล
2. สำนักการศึกษา
3. สำนักคลัง
4. สำนักสาธารณสุขและสิ่งแวดล้อม
5. สำนักช่าง
6. สำนักการประปา
7. กองยุทธศาสตร์และงบประมาณ
8. กองสวัสดิการสังคม
9. กองสารสนเทศภาษีและทะเบียนทรัพย์สิน
10. กองการเจ้าหน้าที่
11. หน่วยตรวจสอบภายใน

### 7.2 Equipment Types (ประเภทครุภัณฑ์)
1. ครุภัณฑ์สำนักงาน
2. ครุภัณฑ์ยานพาหนะและขนส่ง
3. ครุภัณฑ์ไฟฟ้าและวิทยุ
4. ครุภัณฑ์โฆษณาและเผยแพร่
5. ครุภัณฑ์การเกษตร
6. ครุภัณฑ์ก่อสร้าง
7. ครุภัณฑ์การแพทย์
8. ครุภัณฑ์วิทยาศาสตร์
9. ครุภัณฑ์การศึกษา
10. ครุภัณฑ์คอมพิวเตอร์
11. ครุภัณฑ์สนาม
12. ครุภัณฑ์ดนตรีและนาฏศิลป์
13. ครุภัณฑ์กีฬา
14. ครุภัณฑ์โรงงาน
15. ครุภัณฑ์อื่น

### 7.3 Material Types (ประเภทวัสดุ)
1. วัสดุสำนักงาน
2. วัสดุไฟฟ้าและวิทยุ
3. วัสดุงานบ้านงานครัว
4. วัสดุก่อสร้าง
5. วัสดุยานพาหนะและขนส่ง
6. วัสดุเชื้อเพลิงและหล่อลื่น
7. วัสดุวิทยาศาสตร์หรือการแพทย์
8. วัสดุการเกษตร
9. วัสดุโฆษณาและเผยแพร่
10. วัสดุเครื่องแต่งกาย
11. วัสดุการศึกษา
12. วัสดุคอมพิวเตอร์
13. วัสดุสนาม
14. วัสดุดนตรีและนาฏศิลป์
15. วัสดุกีฬา
16. วัสดุเครื่องดับเพลิง
17. วัสดุโรงงาน
18. วัสดุอาวุธยุทธภัณฑ์
19. วัสดุอื่น

## 8. สิทธิ์การใช้งาน (RBAC)

### admin
- Equipment: View, Create, Edit, Delete, Approve
- Materials: View, Create, Edit, Delete, Approve

### manager
- Equipment: View, Create, Edit, Approve
- Materials: View, Create, Edit, Approve

### staff
- Equipment: View, Create, Edit
- Materials: View, Create

### viewer
- Equipment: View
- Materials: View

## 9. System Settings
- `max_image_size = 2097152`
- `max_images_per_equipment = 4`
- `low_stock_alert = true`
- `maintenance_notification = true`
- `withdrawal_auto_approve = false`
- `system_timezone = Asia/Bangkok`
- `date_format = dd/MM/yyyy`
- `currency = THB`

## 10. แนวทาง UI/UX
- เรียบง่าย
- ชัดเจน
- ใช้งานเร็ว
- รองรับ PC / Tablet / Mobile
- แบบฟอร์มกรอกข้อมูลสั้นและเข้าใจง่าย
- มี search + filter หลายเงื่อนไข
- ปุ่มสำคัญมองเห็นชัด

## 11. ข้อกำหนดสำคัญ
- รหัสครุภัณฑ์ต้องตรงรูปแบบเสมอ
- ข้อมูลสำคัญต้อง validate ฝั่ง backend
- ระบบต้องรองรับ audit log
- สต๊อกวัสดุต้องไม่ติดลบ
- สิทธิ์ต้องตรวจทั้ง frontend และ backend
